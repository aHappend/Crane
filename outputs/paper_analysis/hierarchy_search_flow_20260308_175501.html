Crane: Hierarchical Block Search

Core Point

The paper does not directly optimize a single number called depth. It optimizes a hierarchical block structure H plus state workloads {s_i}. Depth is only a by-product of the chosen hierarchy tree.

Hierarchy Tree

Top block
  -> Block A
     -> Sub-block A1
        -> Layer 1
        -> Layer 2
     -> Sub-block A2
        -> Layer 3
  -> Block B
     -> Layer 4
     -> Layer 5

Different branches can stop at different depths. Therefore there is no single global best depth like 2 or 3. The true object being optimized is the tree H.

Per-Level State Derivation

For any composite block with N direct children, the paper derives 2N-1 states and constructs ScT and MeT for that block. If one child is still composite, the same procedure can be recursively applied again inside it.

Search Flow

Step A. Start from a coarse hierarchy and optimize schedule on it.
Step B. Pick a block and partition it into finer children.
Step C. Rebuild states, ScT, and MeT for the refined hierarchy.
Step D. Recompute latency, energy, and EDP.
Step E. Keep the refined hierarchy only if it improves cost enough.
Step F. Repeat until no promising refinement remains.

Why This Finds the Best Depth Indirectly

  • A branch that keeps being refined becomes deeper.
  • A branch that stops early stays shallow.
  • The final tree may have mixed depths across branches.

So the paper finds the best depth profile indirectly by finding the best hierarchy tree.

What Figure 3 Means

Figure 3 implies that scheduling is hierarchical: there is top-level block scheduling, and inside a block its own children also have pipeline format and derived states. Therefore block-internal pipeline scheduling is part of the method.

Current Gap With Our Code

  • We already have block-level ScT/MeT and approximate hierarchy refinement.
  • We still do not have the full recursive child-level ScT/MeT derivation required by the paper.
  • So our current implementation is hierarchy-aware, but not yet full paper-grade recursive hierarchy scheduling.
